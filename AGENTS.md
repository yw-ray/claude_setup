# Personal Claude/Cursor Settings AGENTS.md

이 문서는 개인 환경(`~/.claude/`)에서 AI 에이전트가 따라야 할 규칙과 가이드라인을 제공합니다.

## 용어 정리

### 개인 설정 (Personal Settings)

**정의**: 개발자 개인의 작업 방식, 선호사항, 워크플로우를 정의하는 설정

**위치**: `~/.claude/` 디렉토리 (홈 디렉토리 기준)

**특징**:
- 개발자별로 다를 수 있음
- 프로젝트 코드베이스와 독립적
- 버전 관리에 포함되지 않음 (개인 Git 저장소로 별도 관리 가능)
- 여러 프로젝트에서 공통으로 사용 가능

**예시**:
- 개인 명령어 (`~/.claude/commands/`)
- 개인 작업 규칙 (`~/.claude/rules/`)
- 개인 스킬 문서 (`~/.claude/skills/`)

### 공용 설정 (Shared Settings)

**정의**: 프로젝트 팀 전체가 공유하는 AI 에이전트 설정 및 가이드 문서

**위치**: 프로젝트 코드베이스의 `.claude/` 또는 `.cursor/` 디렉토리 (레포 설정)

**특징**:
- 팀 전체가 공유하는 정보
- 버전 관리에 포함됨
- 프로젝트별로 다를 수 있음
- 프로젝트 코드베이스의 일부 (레포 설정)

**예시**:
- 팀 공통 스킬 문서 (`<repo>/.claude/skills/`)
- 팀 공통 작업 계획 (`<repo>/.claude/plans/`)
- 프로젝트 가이드 문서 (`<repo>/AGENTS.md`)
- 레포 설정 (`<repo>/.cursor/` 또는 `<repo>/.claude/`)

**참고**: `mats-monorepo`는 여러 프로젝트 중 하나의 예시일 뿐입니다. 다른 프로젝트(예: `other-project/.claude/`, `other-project/.cursor/`)도 동일한 원칙이 적용됩니다.

## 개인 설정과 공용 설정 구분

### 개인 환경 (`~/.claude/`)

개인 환경은 각 개발자의 작업 방식과 선호사항을 반영하는 설정을 저장합니다. **절대 프로젝트 코드베이스에 포함되어서는 안 됩니다.**

**개인 환경에 포함되는 항목:**
- `~/.claude/commands/`: 개인 명령어 (예: `/weekly`, `/finish` 등)
- `~/.claude/rules/`: 개인 작업 규칙
- `~/.claude/skills/`: 개인 스킬 문서

**중요 규칙:**
- ❌ **절대 프로젝트 코드베이스(예: `mats-monorepo/.claude/commands/` 등)에 개인 설정을 생성하지 마세요**
- ✅ 개인 명령어나 규칙은 항상 `~/.claude/` 경로에 생성하세요
- ✅ 프로젝트 코드베이스의 `.gitignore`에 `.claude/commands/`가 포함되어 있어도, 개인 설정은 코드베이스에 두지 않습니다

### 공유 설정 (프로젝트 코드베이스)

프로젝트 코드베이스의 `.claude/` 또는 `.cursor/` 디렉토리는 팀 전체가 공유하는 레포 설정을 포함합니다.

**공유 설정에 포함되는 항목:**
- `<repo>/.claude/skills/`: 팀 공통 스킬 문서
- `<repo>/.claude/plans/`: 팀 공통 작업 계획 문서
- `<repo>/.cursor/`: 레포 설정 (프로젝트별 Cursor 설정)
- `<repo>/AGENTS.md`: 프로젝트 전체 가이드 문서

**중요 규칙:**
- ✅ 공유 설정은 버전 관리에 포함됩니다
- ✅ 모든 팀원이 공유하는 정보만 포함합니다
- ❌ 개인 작업 방식이나 선호사항은 포함하지 않습니다

## 경로 참조 규칙

### 개인 환경 문서에서의 경로 참조

개인 환경(`~/.claude/`)의 문서에서 프로젝트 코드베이스의 파일을 참조할 때:

- ✅ **프로젝트 코드베이스 경로**: `<repo-name>/.claude/skills/...` 또는 `<repo-name>/.cursor/...` 형식으로 명시
- ✅ **저장소 이름 명시**: 로컬 경로처럼 보이지 않도록 저장소 이름을 포함
- 예: `mats-monorepo/.claude/skills/confluence-api-access/SKILL.md`

### 프로젝트 코드베이스 문서에서의 경로 참조

프로젝트 코드베이스의 문서에서:

- ✅ **프로젝트 내부 경로**: `.claude/skills/...` 형식으로 참조 가능
- ❌ **개인 경로 참조 금지**: `~/.claude/...` 경로는 프로젝트 문서에서 참조하지 않음

## 개인 명령어 (`~/.claude/commands/`)

개인 명령어는 개발자별 워크플로우를 자동화하는 명령어입니다.

**현재 개인 명령어:**
- `/weekly`: 주간 보고서 자료 수집 및 배치 workflow
- `/finish`: 커밋 전 마무리 작업 수행
- `/new`: 새 작업 시작
- `/fork`: 브랜치 포크
- `/review`: 코드 리뷰
- `/feedback`: 피드백 처리
- `/sync`: 동기화
- `/switch`: 브랜치 전환
- `/revert-fork`: 포크 되돌리기

**명령어 작성 규칙:**
- 각 명령어는 `~/.claude/commands/<command-name>.md` 형식으로 저장
- 명령어는 특정 프로젝트에 종속될 수 있으며, 이 경우 `**Repository**: <repo-name>` 섹션에 명시
- 프로젝트 코드베이스의 파일을 참조할 때는 저장소 이름을 포함한 전체 경로 사용

## 실수 방지 체크리스트

AI 에이전트가 개인 설정을 생성할 때 다음을 확인하세요:

- [ ] 생성하려는 파일이 개인 설정인가? (`commands/`, `rules/` 등)
- [ ] 현재 작업 디렉토리가 프로젝트 코드베이스인가?
- [ ] 개인 설정이라면 `~/.claude/` 경로에 생성하는가?
- [ ] 프로젝트 코드베이스에 개인 설정을 생성하려고 하지 않는가?

**만약 실수로 프로젝트 코드베이스에 개인 설정을 생성했다면:**
1. 즉시 해당 파일을 `~/.claude/`로 이동
2. 프로젝트 코드베이스에서 삭제
3. `.gitignore`에 해당 경로가 포함되어 있는지 확인
