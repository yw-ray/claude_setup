# Test Driven Development (TDD) Workflow

모든 개발 작업은 Test Driven Development (TDD) 방식으로 수행합니다.

## TDD 사이클

1. **Red (실패하는 테스트 작성)**
   - 새로운 기능을 구현하기 전에 먼저 실패하는 테스트를 작성합니다.
   - 테스트는 기대하는 동작을 명확히 정의해야 합니다.
   - 테스트가 실패하는 것을 확인합니다 (Red 단계).

2. **Green (테스트 통과)**
   - 테스트를 통과하는 최소한의 코드만 작성합니다.
   - 테스트가 통과하는 것을 확인합니다 (Green 단계).

3. **Refactor (리팩토링)**
   - 테스트가 통과한 후 코드를 개선합니다.
   - 리팩토링 후에도 모든 테스트가 통과하는지 확인합니다.

## TDD 적용 원칙

- **기능 구현 전 테스트 작성**: 새로운 기능을 추가하거나 기존 기능을 수정할 때, 먼저 테스트를 작성합니다.
- **테스트 우선**: 구현 코드보다 테스트 코드를 먼저 작성합니다.
- **점진적 개발**: 작은 단위로 테스트를 작성하고 구현합니다.
- **테스트 커버리지**: 핵심 로직에 대한 테스트를 작성합니다.

## 예외 상황

다음과 같은 경우에는 TDD를 엄격하게 따르지 않아도 됩니다:
- 프로토타입이나 탐색적 개발 (spike)
- 단순한 설정 파일 수정
- 문서화 작업
- 리팩토링 작업 (기존 테스트가 있는 경우)

## 테스트 작성 가이드

### Django 프로젝트
- `TestBaseClass`를 상속하는 테스트 클래스 사용
- 테스트 클래스 이름은 `TestClass` suffix 사용 (예: `MyFeatureTestClass`)
- `mango_test_mgmt.settings.local`를 세팅 모듈로 사용
- `@patch`로 외부 의존성 모킹

### Test Runner 프로젝트
- `TestCaseBaseClass`를 상속하는 테스트 클래스 사용
- pytest를 사용하여 테스트 실행
- DRY_RUN 모드로 자동 설정

## 작업 흐름

1. **요구사항 분석**: 구현할 기능의 요구사항을 명확히 이해합니다.
2. **테스트 작성**: 요구사항을 기반으로 실패하는 테스트를 작성합니다.
3. **테스트 실행**: 테스트가 실패하는지 확인합니다 (Red).
4. **구현**: 테스트를 통과하는 최소한의 코드를 작성합니다.
5. **테스트 실행**: 테스트가 통과하는지 확인합니다 (Green).
6. **리팩토링**: 코드를 개선하고 테스트가 여전히 통과하는지 확인합니다.

## AI Assistant 가이드

AI Assistant는 다음을 수행해야 합니다:
- 사용자가 기능 구현을 요청할 때, 먼저 테스트를 작성하도록 제안합니다.
- 테스트 코드와 구현 코드를 함께 제공합니다.
- 테스트가 먼저 작성되고, 그 다음 구현 코드가 작성되는 순서를 유지합니다.
- 테스트 실행 명령어를 제공하여 사용자가 테스트를 확인할 수 있도록 합니다.
